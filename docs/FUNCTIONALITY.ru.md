# Возможности и как ими пользоваться

Простым языком: что умеет сервер и как этим пользоваться. Часть «Для пользователей» можно
давать новичкам как есть. Часть «Для администратора» — про управление.

В примерах домен — `example.com`; замените на свой. Адрес пользователя выглядит как `@имя:example.com`.

---

# Для пользователей

## 1. Вход и клиенты
- **Браузер:** `https://elementweb.example.com` (Element Web) — ничего ставить не нужно.
- **Телефон/ПК:** приложение **Element** (iOS, Android, Windows, Mac, Linux). При входе указать сервер
  `https://matrix.example.com`.
- **Логин** — ваше имя и пароль. Адрес-идентификатор: `@имя:example.com`.

## 2. Регистрация
Регистрация **закрытая** — нужен одноразовый токен-приглашение от администратора. Попросите токен у
`@admin:example.com`, при регистрации введите его. Почта/СМС-подтверждение не требуются.

## 3. Поиск людей и пространства
- **Поиск:** в Element сверху «New message» / поиск — видно **всех** пользователей сервера, ищите по
  имени. Никаких «добавить в контакты» не нужно.
- **Пространства (слева):** комнаты сгруппированы в пространства **«Семья»** и **«Друзья»** —
  как папки. Это не мешает поиску: людей всё равно видно всех. В какое пространство вы входите — решает
  администратор (приглашает вас в нужное).

## 4. Звонки и видео
- В любой комнате — **кнопка звонка** (📞/🎥) сверху. Откроется **Element Call**: голос, видео, **демонстрация экрана**.
- Работает **из браузера и из приложения**, в т.ч. вне домашней сети.
- В одном звонке — **до 10 участников**.
- **Гости (без регистрации):** можно поделиться ссылкой на звонок (в Element Call кнопка пригласить/share link) —
  человек откроет её в браузере, введёт имя и зайдёт **без аккаунта**. Удобно для созвонов с теми, кого нет на сервере.

## 5. Шифрование
- **Личные диалоги шифруются автоматически** (E2EE) — отдельно включать не надо.
- При первом входе с нового устройства Element попросит **подтвердить сессию** (через другое ваше
  устройство или **ключ восстановления**). Сохраните ключ восстановления — без него история
  зашифрованных чатов на новом устройстве не откроется.
- Боты, которым это нужно (AI-ассистент, переводчик), умеют работать в зашифрованных комнатах.

## 6. AI-ассистент (baibot)
Напишите в личку **`@baibot:example.com`**.
- Отвечает **по-русски** по умолчанию; язык можно сменить, просто попросив («reply in English»).
- **Понимает картинки** — пришлите фото, спросите «что на фото?» / «переведи текст с фото».
- Помогает с вопросами; по вопросам про **аккаунт/сервер/доступ** сразу направит к администратору `@admin:example.com`.
- **Диалог сразу зашифрован**, ничего настраивать не нужно.
- В групповой комнате, куда добавлен бот, можно обращаться к нему упоминанием.

## 7. Переводчик
Позовите **`@bot.maubot:example.com`** в комнату (или в личку) и пишите команду:
```
!tr en привет, как дела
```
→ бот ответит переводом на английский. Вместо `en` — любой язык (`ru`, `de`, `fr`, `es`, `tr`, `zh`…).
Исходный язык определяется автоматически. Работает и в зашифрованных комнатах.

## 8. Напоминания
Напишите **`@bot.matrix-reminder-bot:example.com`**:
```
!remindme 2h; забрать заказ
!remindme tomorrow 9am; позвонить маме
!remindme every monday 10:00; планёрка
```
Бот напишет вам в нужный момент. `!listreminders` — список, `!cancelreminder <id>` — отменить.

## 9. Мосты — связать другой мессенджер
Можно завести **свои** Telegram/WhatsApp/и т.д. в Matrix: переписка из них будет приходить и
отправляться прямо из Element. Подключаете **сами**, без одобрения админа. Привязка возможна
**только для пользователей этого сервера** (чужие homeserver’ы привязать свои аккаунты не могут).

Напишите нужному боту в личку команду — он пришлёт инструкцию:

| Сервис | Бот | Как привязать (кратко) |
|---|---|---|
| **Telegram** | `@telegrambot:example.com` | `login` → ввести номер телефона и код из Telegram |
| **WhatsApp** | `@whatsappbot:example.com` | `login` → отсканировать QR в WhatsApp → «Связанные устройства» |
| **Signal** | `@signalbot:example.com` | `login` → отсканировать QR в Signal → «Связанные устройства» |
| **Discord** | `@discordbot:example.com` | `login qr` (или по токену) |
| **Instagram** | `@instagrambot:example.com` | `login` → логин/пароль Instagram |
| **Android-SMS** | `@gmessagesbot:example.com` | `login` → связать с Google Messages (QR), нужен Android-телефон онлайн |

После привязки ваши диалоги из мессенджера появятся в Element. Чтобы отвязать — команда `logout`.
Сообщения и звонки/инвайты вам **от пользователей с других Matrix-серверов** приходят как обычно —
это не мосты, а штатная федерация, она включена.

## 10. Push-уведомления
Уведомления на телефон работают через собственный сервер **ntfy** (`push.example.com`) — отдельно
настраивать обычно не нужно, Element подхватывает его сам.

## 11. Нужна помощь
- Напишите боту поддержки **`@honoroit:example.com`** — ваше сообщение попадёт админам, ответят в этой же переписке.
- Или напрямую администратору **`@admin:example.com`**.
- При входе новые пользователи автоматически попадают в комнату **«Объявления»** — там закреплено приветствие с этой же шпаргалкой.

# Для администратора

## 12. Админ-панель
- **Element Admin (Ketesa / synapse-admin):** `https://admin.example.com` — пользователи, комнаты,
  устройства, удаление комнат, медиа и т.д. Вход под админ-аккаунтом.
- Внутри Element комнатами/правами тоже можно управлять (вы — owner ключевых комнат).

## 13. Выдача приглашений (регистрация по токену)
Регистрация закрыта и идёт через **MAS** (Matrix Authentication Service). Токены-приглашения
выдаются в аккаунт-панели MAS (`https://matrix.example.com/account/`) под админом, либо через
`@bot.matrix-registration-bot:example.com`. Передайте токен новому пользователю — он введёт его при регистрации.

## 14. Модерация (Draupnir)
Бот **`@bot.draupnir:example.com`** уже создал **комнату управления** «@admin's Draupnir» и пригласил вас.
Команды пишутся **в этой комнате**:
- `!draupnir status` — статус.
- `!draupnir rooms add <room>` — взять комнату под защиту.
- `!draupnir ban <user> <list> <причина>` — бан, `!draupnir unban …`.
- Подписка на готовые списки банов, антиспам-защиты и т.д. — см. документацию Draupnir.

Это «рубильник безопасности»: при необходимости быстро забанить/выгнать. Для закрытого сервера
нагрузки почти нет, но защита под рукой.

## 15. Хелпдеск (Honoroit)
Пользователи пишут `@honoroit:example.com` — каждое обращение появляется **тредом** в backoffice-комнате
«Поддержка», куда приглашён админ. Отвечаете в треде — ответ уходит пользователю. Бот уже в своей комнате.

## 16. Объявления / приветствие
Комната **«Объявления»** (`#announcements:example.com`): новые пользователи **авто-вступают** в неё,
видят закреплённое приветствие. Писать может **только админ** (broadcast-only), реакции разрешены.
Чтобы сделать объявление — просто напишите в эту комнату.

## 17. Пространства «Семья» / «Друзья»
Созданы два пространства. Добавляйте в них комнаты (внутри пространства «Add room») и приглашайте
людей в нужное пространство — так получается сегментация по группам, при этом глобальный поиск
людей продолжает работать.

## 18. Xray — обход блокировок (hot-swap)
Управление с сервера, командой `xray-manage` (см. основной README):
```
xray-manage sub '<ссылка-на-подписку>'     # задать подписку
xray-manage link 'vless://...'             # одиночная ссылка
xray-manage refresh | status | test | on | off
```
Можно менять апстрим **без пересборки Matrix**. Какие домены гнать через VLESS — список
`/usr/local/etc/xray/swap/proxy-domains.txt`, потом `xray-manage refresh`. Если все апстримы умрут —
трафик автоматически пойдёт напрямую (сервер не «ляжет»).

## 19. Бэкапы и обслуживание
- **Postgres** — автоматические логические дампы (`postgres-backup`); снаружи их забирает Proxmox PBS.
- **Весь LXC** — снапшоты Proxmox/ZFS + PBS.
- **БД Synapse** ужимается `synapse-auto-compressor` (state-groups).
- **Логи:** `journalctl -u matrix-synapse.service -f` (и аналогично для других контейнеров).
- **Обновление:** `just update && just install-all`.

## 20. Удаление комнат
Удаляйте через Element Admin. Комнаты, покинутые всеми участниками, **вычищаются из БД примерно
за час** (а не за 28 дней) — поэтому в админке больше не висит «Deletion in progress».
